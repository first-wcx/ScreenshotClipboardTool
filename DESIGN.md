# Integrated Capture Clipboard 方案

## 目标

新建一个独立应用，把截图工具和剪切板查看器整合到一个程序里，同时不修改现有 `screenshot_tool` 和 `剪切板` 两个目录。新目录内保留必要源码副本，因此源码文件夹可以单独拎走运行或打包。

当前版本优先解决常驻内存、体积和主流程体验：
- 一个常驻程序负责剪切板历史、网络同步、区域截图、图片编辑、贴图和截图历史。
- 暂时砍掉 OCR 和滚动长截图入口，避免常驻资源和打包资源膨胀。
- 主窗口只保留高频按钮，其他操作留在右键菜单或设置页。
- 截图编辑器按需加载，启动时不导入 `screenshot_tool.app`。
- 主窗口默认尺寸保持 `430x560`，截图历史页只调整内部布局。

## 功能结构

主窗口分成 3 个标签页：
- 剪切板：文本、文件、图片历史，复制、打开、编辑，右键菜单保留删除和定位等操作。
- 截图：区域截图复制、区域截图编辑、区域截图贴图，保留最近截图列表。
- 同步/设置：保存目录、网络同步配置、可修改全局热键。

托盘菜单：
- 剪切板
- 历史截图
- 设置
- 退出

默认热键：
- `Alt+A` 截图复制
- `Alt+S` 截图编辑
- `Alt+D` 截图贴图
- `Ctrl+Shift+M` 显示主窗口

设置项：
- 截图前是否隐藏主窗口
- 开机自启
- 保存目录、网络同步和热键

## 性能设计

常驻只启动：
- 一个 Tk 主线程
- 一个剪切板轮询定时器
- 一个托盘线程
- 网络同步只在启用后启动

懒加载：
- 截图遮罩、图片编辑器、贴图窗口只在第一次截图/编辑/贴图时加载。
- 截图历史缩略图只加载当前选中项。
- 图片预览只读取当前需要显示的图片，不批量解码全部图片。

剪切板历史：
- 用 Windows `GetClipboardSequenceNumber()` 判断变化，未变化时不读取剪切板。
- 图片记录只保存文件路径，不把图片二进制塞进 JSON。
- 大图片只在预览、编辑或同步时读取。

图片编辑：
- 编辑器内部用 Pillow 原图作为数据源，Tk 只持有当前缩放后的显示图。
- 大图默认按窗口缩小显示，避免从剪切板入口打开时过度放大。
- 撤销栈沿用截图工具实现，避免重复维护两套编辑器。

## 体积控制

依赖保持最小：
- 必需：Pillow、pystray
- 标准库：tkinter、ctypes、socket、threading、json、pathlib
- 不加入 customtkinter、opencv、numpy、transformers、easyocr、torch 等大型包

PyInstaller：
- 使用 onefile 打包。
- 只按需隐藏导入截图编辑器相关模块，不递归收集整个截图包。
- 不打包 OCR PowerShell 脚本和大图标资源。

## 数据目录

统一使用：

```text
%APPDATA%/IntegratedCaptureClipboard/
  settings.json
  clipboard_history.json
  clipboard_sync.json
  images/
  captures/
  thumbnails/
```

## 验收标准

功能：
- 单开一个程序即可完成剪切板历史、截图、编辑、复制、贴图、同步核心工作流。
- 热键可以在设置页修改并重新注册。

性能：
- 空闲 CPU 接近 0。
- 启动阶段不加载截图编辑器主模块。
- 常驻内存低于旧整合版，截图/编辑时才允许临时上升。

体积：
- exe 不引入大型机器学习/图像处理依赖。
- 打包产物保持 onefile 便携形态。

兼容：
- Windows 10/11。
- 高 DPI 下主界面、列表、编辑器不模糊、不裁字。
